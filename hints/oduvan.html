Can you explain functions using this particular task?

With the big pleasure. This is why we have hints on CheckiO

Imagine you need to multiply two numbers and print(output) the result so you can see it in the console. For example, 5 and 6.

print(5*6)

Output:

30

Now instead of 5 and 6, we can use variables. For what? You will see in the further examples.

a = 5
b = 6
print(a*b)

Output:

30

You can change variable a, run the script again, and see the different result

a = 6
b = 6
print(a*b)

Output:

36

Or you can copy your code twice, change variables and see the result of two multiplications

a = 5
b = 6
print(a*b)


a = 6
b = 6
print(a*b)

Output:

30
36

Now I can show you how the function can simplify the above code.

def mult_two(a, b):
    return a*b

print(mult_two(5, 6))
print(mult_two(6, 6))

Output:

30
36

As you can see, function mult_two now holds all of the logic, and you only change the arguments to see different results.

Awesome. Thank you, but can you please explain the structure of the function in more detail so I can freely use it in the other CheckiO missions

Of course. Here is the function once again:

def mult_two(a, b):
    return a*b

Always starts with def

The name of the function mult_two - you will use the name to call the function in future

Two arguments (a, b) - those are variables that will get their values when the function is called. For example, when we call the function mult_two(5, 6) - inside the function, we set variable a to 5 and variable b to 6.

You can also see function definitions with type hints.

def mult_two(a: int, b: int) -> int:
    return a*b

Don’t be confused by : int and -> int. They explain to the user (someone who reads the code) the expected type of arguments and the type of the result of the function call. But it is ok to ignore those for now.

Now the block of the function that starts with four spaces and return says what should be the result of the function calls return a*b

In our example print(mult_two(5, 6)) - the result of the multiplication of two values 5 and 6 will be returned and use ad the first argument for the function print

If you want your function to have more than one line of code in it - all of the lines should have the first 4 spaces. For example, lets use one more variable c, where we put the result of the multiplication and return the result

def mult_two(a, b):
    c = a*b
    return c

Ok. Thank you, now it is clear enough. But when I have a correct function, what should I do next?

On top of the code editor, you have two buttons, “run code” and “check solution”.

When you hit the first one - it runs the code from code editor. When you use “check solution” - the system checks your function and shows the “congratulations” panel when it is working as expected.

When something doesn’t work as expected - the panel will show you the call of the function for which is return the wrong value
