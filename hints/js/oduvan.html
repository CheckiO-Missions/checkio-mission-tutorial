With the big pleasure. This is why we have hints on CheckiO!

Imagine you need to multiply two numbers and print(output) the result so you can see it in the console. For example, 5 and 6.

console.log(5*6);

Output:

30

Now instead of 5 and 6, we can use variables. For what? You will see in the further examples.

let a = 5;
let b = 6;
console.log(a*b);

Output:

30

You can change variable a, run the script again, and see the different result

let a = 6;
let b = 6;
console.log(a*b);

Output:

36

Or you can copy your code twice (without let in the copy), change variables and see the result of two multiplications

let a = 5;
let b = 6;
console.log(a*b);


a = 6;
b = 6;
console.log(a*b);

Output:

30
36

Now I can show you how the function can simplify the above code.

function multTwo(a, b) {

    return a*b;
}

console.log(multTwo(5, 6));
console.log(multTwo(6, 6));

Output:

30
36

As you can see, function multTwo now holds all of the logic, and you only change the arguments to see different results.

Of course. Here is the function once again:

function multTwo(a, b) {

    return a*b;
}

Always starts with function.

The name of the function multTwo - you will use the name to call the function in future.

Two arguments (a, b) - those are variables that will get their values when the function is called. For example, when we call the function multTwo(5, 6) - inside the function, we set variable a to 5 and variable b to 6.

You can also see function definitions with type hints.

function multTwo(a: number, b: number): number {

    return a*b;
}

Don’t be confused by : number. It explains to the user (someone who reads the code) the expected type of arguments and the type of the result of the function call. But it is ok to ignore those for now.

Now the block of the function that inside of { and } and return says what should be the result of the function calls return a*b.

In our example console.log(multTwo(5, 6)) - the result of the multiplication of two values 5 and 6 will be returned and use ad the first argument for the function console.log().

You function can have more than one line of code. For example, lets use one more variable c, where we put the result of the multiplication and return the result.

function multTwo(a, b) {

    let c = a*b;
    
    return c;
}

On top of the code editor, you have two buttons, “run code” and “check solution”.

When you hit the first one - it runs the code from code editor. When you use “check solution” - the system checks your function and shows the “congratulations” panel when it is working as expected.

When something doesn’t work as expected - the panel will show you the call of the function for which is return the wrong value
