Imaginez que vous devez multiplier deux nombres et imprimer (sortir) le résultat pour le voir dans la console. Par exemple, 5 et 6.

console.log(5*6) ;

Résultat :

30

Maintenant, au lieu de 5 et 6, nous pouvons utiliser des variables. Pour quoi faire ? Vous le verrez dans les exemples suivants.

let a = 5 ;
let b = 6 ;
console.log(a*b) ;

Résultat :

30

Vous pouvez changer la variable a, exécuter le script à nouveau et voir le résultat différent

let a = 6 ;
let b = 6 ;
console.log(a*b) ;

Résultat :

36

Vous pouvez également copier votre code deux fois (sans le laisser dans la copie), changer les variables et voir le résultat de deux multiplications

let a = 5 ;
let b = 6 ;
console.log(a*b) ;


a = 6 ;
b = 6 ;
console.log(a*b) ;

Sortie :

30
36

Je vais maintenant vous montrer comment la fonction peut simplifier le code ci-dessus.

function multTwo(a, b) {

    renvoie a*b ;
}

console.log(multTwo(5, 6)) ;
console.log(multTwo(6, 6)) ;

Sortie :

30
36

Comme vous pouvez le constater, la fonction multTwo contient désormais toute la logique, et il vous suffit de modifier les arguments pour obtenir des résultats différents.

C'est génial. Merci, mais pourriez-vous expliquer la structure de la fonction plus en détail afin que je puisse l'utiliser librement dans les autres missions CheckiO.

Bien entendu. Voici à nouveau la fonction :

function multTwo(a, b) {

    return a*b ;
}

Le nom de la fonction commence toujours par function.

Le nom de la fonction multTwo - vous utiliserez ce nom pour appeler la fonction à l'avenir.

Deux arguments (a, b) - il s'agit de variables qui obtiendront leur leurs valeurs lors de l'appel de la fonction. Par exemple, lorsque nous appelons la fonction multTwo(5, 6) - à l'intérieur de la fonction, nous attribuons la valeur 5 à la variable a et la valeur 6 à la variable b.

Vous pouvez également voir les définitions de fonctions avec des indications de type.

function multTwo(a : number, b : number) : number {

    return a*b ;
}

Ne vous laissez pas déconcerter par : number. Il explique à l'utilisateur (celui qui lit le code) le type attendu des arguments et le type du résultat de l'appel de la fonction. Mais vous pouvez les ignorer pour l'instant.

Maintenant, le bloc de la fonction qui se trouve à l'intérieur de { et } et de return dit ce que doit être le résultat de l'appel de la fonction return a*b.

Dans notre exemple, console.log(multTwo(5, 6)) - le résultat de la multiplication de deux valeurs 5 et 6 sera retourné et utilisé comme premier argument de la fonction console.log().

Votre fonction peut comporter plus d'une ligne de code. Par exemple, nous pouvons utiliser une variable supplémentaire c, dans laquelle nous plaçons le résultat de la multiplication et renvoyons le résultat.

function multTwo(a, b) {

    let c = a*b ;
    
    retourne c ;
}
Ok. Merci, c'est maintenant assez clair. Mais lorsque j'ai une fonction correcte, que dois-je faire ensuite ?

En haut de l'éditeur de code, vous avez deux boutons, "exécuter le code" et "vérifier la solution".

Lorsque vous appuyez sur le premier, le code est exécuté à partir de l'éditeur de code. Lorsque vous utilisez "vérifier la solution", le système vérifie votre fonction et affiche le panneau "félicitations" lorsqu'elle fonctionne comme prévu.

Lorsque quelque chose ne fonctionne pas comme prévu, le panneau vous montrera l'appel de la fonction pour laquelle la valeur renvoyée est erronée

ou lorsqu'il y a une erreur dans l'assert
