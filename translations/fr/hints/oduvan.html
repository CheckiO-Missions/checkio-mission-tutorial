Pouvez-vous expliquer les fonctions à l'aide de cette tâche particulière ?

Avec grand plaisir. C'est la raison pour laquelle nous avons des astuces sur CheckiO

Imaginez que vous devez multiplier deux nombres et imprimer (sortir) le résultat pour le voir dans la console. Par exemple, 5 et 6.

print(5*6)

Résultat :

30

Maintenant, au lieu de 5 et 6, nous pouvons utiliser des variables. Pour quoi faire ? Vous le verrez dans les exemples suivants.

a = 5
b = 6
print(a*b)

Résultat :

30

Vous pouvez changer la variable a, exécuter à nouveau le script et voir le résultat différent

a = 6
b = 6
print(a*b)

Résultat :

36

Vous pouvez également copier votre code deux fois, changer les variables et voir le résultat de deux multiplications

a = 5
b = 6
print(a*b)


a = 6
b = 6
print(a*b)

Résultat :

30
36

Je vais maintenant vous montrer comment la fonction peut simplifier le code ci-dessus.

def mult_two(a, b) :

    return a*b

print(mult_two(5, 6))
print(mult_two(6, 6))

Sortie :

30
36

Comme vous pouvez le constater, la fonction mult_two contient désormais toute la logique, et il vous suffit de modifier les arguments pour obtenir des résultats différents.

C'est formidable. Merci, mais pourriez-vous expliquer la structure de la fonction plus en détail afin que je puisse l'utiliser librement dans les autres missions CheckiO ?

Bien entendu. Voici une image montrant la structure de la fonction :

Et voici à nouveau notre fonction :

def mult_two(a, b) :

    return a*b

Commence toujours par def.

Le nom de la fonction mult_two - vous utiliserez ce nom pour appeler la fonction à l'avenir

Deux arguments (a, b) - il s'agit de variables qui prendront leur valeur lorsque la fonction sera appelée. Par exemple, lorsque nous appelons la fonction mult_two(5, 6) - à l'intérieur de la fonction, nous attribuons la valeur 5 à la variable a et la valeur 6 à la variable b.

Vous pouvez également voir les définitions de fonctions avec des indications de type.

def mult_two(a : int, b : int) -> int :

    return a*b

Ne vous méprenez pas sur les termes : int et -> int. Ils expliquent à l'utilisateur (celui qui lit le code) le type attendu des arguments et le type du résultat de l'appel de la fonction. Mais vous pouvez les ignorer pour l'instant.

Le bloc de la fonction qui commence par quatre espaces et return indique le résultat de l'appel de la fonction return a*b.

Dans notre exemple, print(mult_two(5, 6)) - le résultat de la multiplication de deux valeurs 5 et 6 sera retourné et utilisé comme premier argument de la fonction print.

Si vous souhaitez que votre fonction comporte plus d'une ligne de code, toutes les lignes doivent comporter les quatre premiers espaces. Par exemple, nous pouvons utiliser une variable supplémentaire, c, dans laquelle nous plaçons le résultat de la multiplication et renvoyons le résultat

def mult_two(a, b) :

    c = a*b
    
    return c

Ok. Merci, c'est maintenant assez clair. Mais quand j'ai une fonction correcte, que dois-je faire ensuite ?

En haut de l'éditeur de code, vous avez deux boutons, "Exécuter le code" et "Vérifier la solution".

Lorsque vous appuyez sur le premier, le code est exécuté à partir de l'éditeur de code. Lorsque vous utilisez "Vérifier la solution", le système vérifie votre fonction et affiche le panneau "Félicitations" lorsqu'elle fonctionne comme prévu.

Lorsque quelque chose ne fonctionne pas comme prévu, le panneau vous montrera l'appel de la fonction pour laquelle la valeur retournée est incorrecte

ou lorsqu'il y a une erreur dans l'assert
