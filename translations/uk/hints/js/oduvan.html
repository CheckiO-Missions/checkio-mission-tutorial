Ось дуже хороша і коротка стаття, яка пояснює, як працюють функції в JavaScript.

Чи можеш ти пояснити функції, використовуючи конкретно цю місію?

З великим задоволенням! Саме тому ми маємо підказки на CheckiO.

Уяви, що тобі потрібно перемножити два числа і надрукувати (вивести) результат так, щоб побачити його в консолі. Наприклад, 5 і 6.

console.log(5*6);

Вивід:

30

Тепер замість 5 і 6, ми можемо використати змінні. Для чого? Ти побачиш це у наступних прикладах.

let a = 5;
let b = 6;
console.log(a*b);

Вивід:

30

Ти можеш змінити значення змінної a, запустити сценарій знову і побачити інший результат.

let a = 6;
let b = 6;
console.log(a*b);

Вивід:

36

Або ти можеш здублювати свій код (без let у копії), змінити значення змінних і побачити результати двох множень.

let a = 5;
let b = 6;
console.log(a*b);


a = 6;
b = 6;
console.log(a*b);

Вивід:

30
36

Тепер я можу показати тобі, як використання функції спрощує такі операції.

function multTwo(a, b) {
    
    return a*b;
}

console.log(multTwo(5, 6));
console.log(multTwo(6, 6));

Вивід:

30
36

Як ти бачиш, функція multTwo тепер містить всю логіку рішення і ти лише змінюєш значення аргументів, щоб отримати інший результат.

Приголомшливо! Дякую, та чи можеш ти будь ласка пояснити структуру функції більш детально, щоб я міг вільно використовувати це в інших місіях CheckiO?

Звісно! Ось запис функції ще раз:

function multTwo(a, b) {

    return a*b;
}

Завжди починай з function.

Далі йде ім'я функції multTwo - надалі ти будеш використовувати це ім'я для виклику функції.

Два аргументи (a, b) - це змінні, які набувають значень під час виклику функції. Наприклад, коли ти викликаєш функцію multTwo(5, 6) - всередині функції змінній a присвоюється значення 5 і змінній b - 6.

Також, ти можеш бачити розширений запис виразу функції:

function multTwo(a: number, b: number): number {

    return a * b;
}

Не лякайся через : number. Вони пояснюють користувачу (тому, хто читатиме код) очікуваний тип аргументу і тип результату виклику функції. Їх використання не обов'язкове, але корисне.

далі блок функції всередині { і } і з return який показує що повинно бути результатом виклику функції: return a*b.

В нашому прикладі console.log(multTwo(5, 6)) - результат множення двох значень 5 і 6 буде повернений і використаний як аргумент для функції console.log().

Твоя функція може мати більше одного рядка коду. Наприклад, введемо ще одну змінну c, якій ми присвоїмо результат множення і повернемо цю змінну.

function multTwo(a, b) {

    let c = a*b;

    return c;
}
Зрозуміло. Дякую тобі, тепер все досить ясно. Але коли я написав коректну функцію, що мені робити далі?

Вгорі редактора коду є дві кнопки - “Run Code” і “Check Solution”.

Коли ти натиснеш першу, вона запустить код із редактора (включно із assert). Коли натиснеш “Check Solution” - система перевірить твою функції рядом тестів і покаже панель вітання “congratulations”, якщо все працює як очікувалось.

Якщо щось працює неочікувано, панель покаже виклик функції, для якого повернуто не вірне значення.

або коли помилка в assert.
